# digest_format — контракт дайджеста

Контракт между конвейером (пишет) и доставкой (читает, [docs/telegram_bot.md](../../../../docs/telegram_bot.md)). Прогон перезаписывает файл `digest_latest.md` в этой папке; история — файлами `ГГГГ-ММ-ДД_ЧЧММ.md` по желанию.

Front-matter — машиночитаемая часть: доставка берёт из неё `run_id` (ключ идемпотентности — повторный запуск не шлёт дубль), фаза 2 соберёт из `items[]` inline-кнопки, не трогая конвейер. `id` — из журнала откликов, короткий (`JV-####`): лимит `callback_data` в Telegram — 64 байта. Тело — человекочитаемые 5–10 строк, разметка только `[текст](url)`, `**жирный**`, `# заголовок`.

Тело — блок на вакансию, а не строка: читается с телефона. Роль — жирная ссылка; вторая строка — компания, формат, деньги, ключевой фильтр; третья — совпадение и почему. Внутренние ID (`JV-####`) в тело не выносятся — они живут в front-matter и на кнопках.

Поля items[]: `source` — hh | linkedin | telegram (по умолчанию hh); `contact` — @username автора поста для Telegram-вакансий: карточка получает кнопку «Написать автору» вместо «Открыть», а тело — заготовку первого сообщения. Дайджест делится на секции по источникам.

```markdown
---
run_id: [ГГГГ-ММ-ДД-ЧЧММ]
stats: найдено [N] · порог [M] · серая зона [K]
items:
  - {id: JV-0001, title: "[Роль — Компания]", url: "[...]", fit: [NN], zone: work, source: hh}
  - {id: JV-0002, title: "[Роль — Компания]", url: "[...]", fit: [NN], zone: work, source: telegram, contact: "[@автор]"}
---
# Лучшее за [дата]

[1–2 строки сводки: сколько нашли, откуда, что самое горячее]

— HeadHunter —

**[[Роль]]([url])**
🏢 [Компания — чем занимается, одной строкой]
💰 [вилка | «~оценка (рынок)»] · [формат] · [требуемый опыт]
Задачи: [2–3 ключевые задачи роли своими словами]
[▰▰▰▰▱ по десяткам] [NN] % — совпало: [главное из резюме]
Не хватает: [что режет и как закрыть — или «ничего критичного»]

— Telegram — прямой контакт с автором —

**[[Роль]]([url поста])**
🏢 [Компания/автор] · 💰 [вилка] · [формат]
Задачи: [...]
[▰▰▰▰▱] [NN] %
Первое сообщение автору: «[короткий питч 2 предложения под пост]»
```

При сбое доставки рядом появляется `delivery_failures.md` — по строке на недоставленный пуш; следующая сессия агента видит его через слой памяти.
