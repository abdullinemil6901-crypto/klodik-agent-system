# job_log — журнал откликов

Единый журнал конвейера вакансий: дедупликация, статусы, follow-up. Правила из [docs/pipelines.md §2](../../../docs/pipelines.md): одна вакансия — одна строка (включая fit < 50% — иначе дедупликация не работает); отклик без ответа несколько дней — статус `follow-up` и строка в дайджест.

ID стабильные и короткие (`JV-0001`, `JV-0002`, …) — они же уходят в `items[]` дайджеста и в `callback_data` кнопок фазы 2 (лимит Telegram — 64 байта).

| ID | Дата | Компания | Роль | fit% | Канал | Статус | Следующий шаг |
|---|---|---|---|---|---|---|---|
| JV-0001 | `[ГГГГ-ММ-ДД]` | `[...]` | `[...]` | `[NN]` | `[hh / getmatch / telegram / ...]` | `[найдена / к отправке / отправлена / ответ / отказ / follow-up / пропущена]` | `[...]` |

## Статусы

- **найдена** → скоринг выполнен, решение не принято (серая зона).
- **к отправке** → пакет отклика готов, ждёт человека.
- **отправлена** → дата отправки в «Следующий шаг».
- **follow-up** → нет ответа несколько дней, нужно напомнить.
- **пропущена** → решение человека или fit < 50%; строка остаётся для дедупликации.
