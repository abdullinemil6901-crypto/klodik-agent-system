# MEMORY — индекс памяти

Корневой файл vault. Любой агент начинает чтение памяти отсюда.

## Структура

- **00 — Профиль** — кто пользователь и как с ним работать.
  - [[00 — Профиль/user_profile|user_profile]] — базовая информация и контекст.
  - [[00 — Профиль/preferences|preferences]] — стиль ответов, формат, тон.
  - [[00 — Профиль/working_agreements|working_agreements]] — обязательные правила, перекрывают дефолты.
- **01 — Память агента** — что копится по ходу сессий.
  - [[01 — Память агента/feedback_log|feedback_log]] — что пошло хорошо/плохо, выводы.
- **02 — Wishlist и репозитории** — что планируем интегрировать.
- **03 — Skills и MCP** — каталог подключённых скилов и MCP-серверов.
- **04 — Проекты** — по одному файлу/папке на рабочий проект.

## Как этим пользоваться

1. Агент, запущенный в рабочей папке, читает `CLAUDE.md` в её корне — там ссылка на этот vault.
2. Новый важный факт или уточнение предпочтений → обновить соответствующий файл, не создавать дубликат.
3. Периодически прогонять консолидацию: слить дубликаты, убрать устаревшее.
