# CLAUDE.md — точка входа для агента

Этот файл читается автоматически в начале любой сессии Claude Code в этом репозитории. Он же — живой пример паттерна «слой памяти»: агент стартует, уже зная контекст и правила, без пересказа.

## Что это за репозиторий

Публичная витрина архитектуры «Клодик» — персональной multi-agent системы на Claude Code. Рабочая версия (Obsidian vault с профилем, резюме, журналами) живёт локально у пользователя и в репозиторий не попадает.

## Правила работы здесь

- **Никаких личных данных в коммитах.** Резюме, отклики, журналы, профиль, контакты, названия работодателей — не публиковать. `vault/` в `.gitignore` — не убирать его оттуда.
- **Секреты только через env.** Токены и chat_id — в переменных окружения или env-файле вне репозитория; `*.env` в `.gitignore`, в репо — только `*.env.example` с пустыми значениями. В коде и текстах ошибок токенов быть не должно.
- **Перед коммитом примеров, конфигов и экспортов** проверять на реальные chat_id, webhook-URL, токены и названия компаний — публикуются только плейсхолдеры.
- Отвечать на русском. Английский — для команд, путей, имён файлов, ключей конфига.
- Кратко и по делу, без эмоджи. Списки — только где они реально помогают.
- Публикуемые примеры — только обезличенные и только реальные: выдуманные цифры и кейсы запрещены.
- Одна фаза работы = один PR. Косметические правки копятся до ближайшего содержательного PR, отдельный цикл им не полагается.

## Карта репозитория

- `README.md` — обзор системы: задача, архитектура, принципы, стек.
- `docs/architecture.md` — устройство слоёв: память, роутинг, конвейер, доставка.
- `docs/demo_verifier_run.md` — разбор живого прогона adversarial-проверки фактов.
- `docs/pipelines.md` — спецификации двух прикладных конвейеров (радар новостей, подбор вакансий).
- `docs/telegram_bot.md` — спецификация модуля Telegram-доставки: контракт дайджеста, сбои, фаза 2.
- `scripts/` — код на stdlib, без зависимостей: `telegram_digest.py` (push), `telegram_bot.py` (интерактив), `journal_pending.py` (очередь задач журнала для рутин), `telegram.env.example`.
- `routines/` — промты плановых прогонов для cron: вакансии (полный и горячий), радар новостей; регистрация — в `routines/README.md`.
- `tests/` — юнит-тесты (`python3 -m unittest discover tests`).
- `vault-template/` — обезличенный скелет файловой памяти: можно склонировать и заполнить под себя.
