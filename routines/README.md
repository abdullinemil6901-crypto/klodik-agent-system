# Рутины — плановые прогоны конвейеров

Промты для запусков по расписанию. До этой папки cron-механика из
[docs/architecture.md](../docs/architecture.md) была задекларирована, но каждый
прогон стартовал вручную — это ежедневная фиксированная утечка времени
(находка №3 аудита). Здесь прогоны оформлены как версионируемые промты:
правится файл — меняется поведение рутины, без правки crontab.

## Состав

| Файл | Конвейер | Рекомендуемая частота |
|---|---|---|
| [job_run_morning.md](job_run_morning.md) | вакансии, полный прогон | 1×/день утром |
| [job_run_hot.md](job_run_hot.md) | вакансии, «горячий» прогон | 2–3×/день днём |
| [news_radar.md](news_radar.md) | радар новостей и возможностей | старт 1×/день |

Каждый прогон вакансий первым шагом обрабатывает необработанные решения из
журнала (`scripts/journal_pending.py`) — так кнопка «В работу» в Telegram
приводит к пакету отклика без ручного запуска сессии.

## Регистрация (OS cron + headless-режим Claude Code)

Рутина — обычный запуск `claude -p` в рабочей папке (её `CLAUDE.md` указывает
на vault). Пример crontab (пути подставить свои):

```cron
# полный прогон вакансий — 08:00
0 8 * * *   cd <рабочая папка> && claude -p "$(cat routines/job_run_morning.md)" >> ~/.local/state/klodik/routines.log 2>&1
# горячие прогоны — 12:00 и 16:00
0 12,16 * * * cd <рабочая папка> && claude -p "$(cat routines/job_run_hot.md)" >> ~/.local/state/klodik/routines.log 2>&1
# радар новостей — 09:00
0 9 * * *   cd <рабочая папка> && claude -p "$(cat routines/news_radar.md)" >> ~/.local/state/klodik/routines.log 2>&1
```

## Правила экономии

Правила из [docs/pipelines.md](../docs/pipelines.md) обязательны и для рутин:
частота повышается только после замера расхода лимитов; при приближении к
лимитам прогон вакансий деградирует до sourcing+скоринг, радар — до одного
finder-агента. Лог рутин — единственное место диагностики headless-запусков,
поэтому финальная строка каждого прогона обязана попадать в него.
