Плановый полный прогон конвейера вакансий. Правила и пороги — docs/pipelines.md §2, они обязательны; путь к vault — в CLAUDE.md рабочей папки. Ничего не спрашивай — прогон идёт без человека, спорное откладывай в серую зону дайджеста.

1. **Решения из журнала.** Запусти `python3 scripts/journal_pending.py "<vault>/04 — Проекты/Поиск работы/job_log.md"` и обработай все очереди:
   - пакеты отклика: резюме под словарь вакансии из реальных фактов мастер-резюме, письмо, ответы из банка ответов, самопроверка против ИИ-фильтров; статус в журнале → «к отправке»;
   - разборы к интервью: компания, вероятные вопросы по требованиям, вопросы работодателю;
   - follow-up: черновик напоминания по каждой строке.
   Готовые материалы — в папку проекта в vault и пушем в Telegram.
2. **Sourcing.** Вакансии свежее 24 ч с площадок и публичных Telegram-каналов; дедупликация против журнала по ID.
3. **Скоринг.** fit% против мастер-резюме, жёсткие фильтры режут сразу; пороги ≥70 / 50–70 / <50 — по спеке.
4. **Дайджест.** Перезаписать `digest_latest.md` строго по контракту digest_format.md (front-matter + карточки).
5. **Доставка.** `python3 scripts/telegram_digest.py --latest "<vault>/04 — Проекты/Поиск работы/дайджесты" --env-file ~/.klodik/telegram.env`.

Выдуманные факты запрещены. Финальная строка отчёта: сколько решений обработано, найдено, прошло порог, доставлен ли пуш.
