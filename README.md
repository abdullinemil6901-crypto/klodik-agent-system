# Клодик — персональная multi-agent AI-система на Claude Code

Личная операционная система на AI-агентах: долговременная память, конвейер агентов с перекрёстной проверкой фактов, автоматизация рутин. Построена на Claude Code + MCP. Работает ежедневно как реальный рабочий инструмент, не демо.

## Задача

Обычная сессия с AI-ассистентом начинается с нуля: контекст теряется, качество ответов зависит от того, что успел рассказать. Клодик решает это архитектурно — система «помнит» контекст, маршрутизирует задачи между агентами и проверяет факты до того, как они попадут в ответ.

### 1. Слой памяти — «передача контекста бизнеса в Claude»

- **Obsidian vault** как файловая база знаний: профиль пользователя, рабочие соглашения, статусы проектов, каталоги инструментов.
- **CLAUDE.md** в корне рабочей папки — точка входа: любая новая сессия агента читает его автоматически и получает весь контекст без пересказа.
- Агент не только читает память, но и **пишет в неё**: журнал фидбэка, статусы, выводы сессий. Система накапливает опыт.

### 2. Роутинг: single vs multi-agent — экономика токенов

Ключевое инженерное решение: multi-agent даёт прирост качества только на **параллелизуемых** задачах (поиск под разными углами, аудит), но стоит ~15× токенов. На линейных задачах single-agent не хуже и дешевле. Роутинг по типу задачи — это то, что отличает работающую систему от сжигания бюджета.

### 3. Multi-agent конвейер с adversarial-проверкой

Паттерн: `finder(×N углов) → verifier → synthesizer`.

- **Finder-агенты** ищут параллельно под разными углами (например: «технологии» и «рыночный спрос»).
- **Verifier** — агент-скептик с презумпцией «данные неверны, пока не подтверждены вторым независимым источником». Ищет конфликты интересов в источниках, проверяет даты и цифры.
- **Synthesizer** собирает выжившее в результат под контекст пользователя (из слоя памяти).

**Живой результат прогона:** из 6 сырых сигналов рынка verifier отсеял 3 (маркетинговые цифры одной заинтересованной площадки, прайсы от вендоров, устаревшие даты), нашёл независимый источник, которого не было у finder-агентов, и скорректировал завышенный кейс дохода до реалистичного диапазона. Одиночный поиск выдал бы все 6 как «факты». Разбор прогона по шагам — в [docs/demo_verifier_run.md](docs/demo_verifier_run.md).

### 4. Рутины и доставка

- Плановые задачи через cron + headless-режим Claude Code: промты прогонов версионируются в [routines/](routines/).
- Отчёты складываются в vault (Obsidian) — читаются с любого устройства.
- Telegram-модуль — готов целиком ([спека](docs/telegram_bot.md)): push-доставка дайджестов (stdlib-скрипт, 0 токенов) и интерактивный бот — команды, самопривязка чата по /start, inline-кнопки решений по серой зоне с записью в журнал откликов. Дальше — голос и роли.

## Принципы

- **Проверка фактов до вывода.** Ни одна цифра из интернета не попадает в отчёт без второго независимого источника.
- **Память вместо пересказа.** Контекст один раз фиксируется в vault — дальше его знают все агенты.
- **Экономика прежде архитектуры.** Multi-agent — только там, где параллелизм окупает токены.
- **Человек в петле на высоких ставках.** Автономно — рутина; решения и внешние действия — через подтверждение.
